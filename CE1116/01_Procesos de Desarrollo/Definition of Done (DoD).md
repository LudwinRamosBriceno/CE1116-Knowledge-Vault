## 📚 Idea/Concepto 

Es una checklist con el trabajo que debe ser realizado en cada elemento del backlog (historias de usuario) antes de que se considere como listo. Se establece una definición común o un acuerdo cuando algo puede ser considerado como "completado".

## 📌 Puntos Claves (Opcional)
- Algunos elementos comunes para considerar algo "completado" son revisiones de código, de diseño, refactoring, pruebas de rendimiento, pruebas unitarias.
- Esta define el nivel o calidad mínima que el equipo se compromete a lograr (lo mínimo que se puede hacer para considerar que algo está listo), a diferencia de los criterios de aceptación que establece las funcionalidades que se esperan.
